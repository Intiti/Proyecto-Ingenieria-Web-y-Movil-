## 2. Descripción del Problema
Los usuarios del sistema de salud enfrentan esperas de meses para consultas debido a la sobredemanda y la falta de organización en la asignación de citas. Este proyecto busca mitigar la ineficiencia mediante un sistema inteligente de lista de espera y triaje digital.

## 3. Definición de Requerimientos

### Requerimientos Funcionales
| ID | Descripción | Rol |
|----|-------------|-----|
| RF1 | Gestión de bloques horarios y disponibilidad médica. | Admin |
| RF2 | Reserva de citas con filtros de especialidad y médico. | Paciente |
| RF3 | Notificación automática de cupos liberados (Lista de espera). | Paciente |
| RF4 | Formulario de triaje para priorización de citas. | Paciente |
| RF5 | Acceso y visualización de historial de exámenes. | Paciente |
| RF6 | Cancelación de citas con liberación automática de cupos. | Paciente/Admin |
| RF7 | Dashboard de estadísticas de sobredemanda. | Admin |

### Requerimientos No Funcionales
* **RNF1 (Usabilidad):** Diseño responsivo y accesible para web y móvil.
* **RNF2 (Seguridad):** Encriptación de datos sensibles y autenticación segura.
* **RNF3 (Rendimiento):** Tiempos de respuesta menores a 2 segundos en consultas.

## 4. Análisis de Usuarios
* **Paciente:** Busca agilidad, transparencia en la espera y acceso a sus documentos.
* **Administrador:** Necesita optimizar la agenda médica y reducir el ausentismo de pacientes.
